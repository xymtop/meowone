# 团队创建指引

本文档说明如何通过纯 `curl` 命令，从零创建一个完整的智能体团队。适用于新来的 Agent 通过 API 自主完成团队初始化。

**前置条件**：MeowOne 后端已运行于 `http://localhost:8000`。

---

## 流程总览

```
1. 新建提示词（可选，但推荐）
2. 新建 MCP（可选）
3. 新建 Skill（可选）
4. 创建智能体（可创建多个）
5. 创建镜像（将多个智能体打包）
6. 创建调度配置（选择调度策略）
7. 创建执行环境（可选，默认 native）
8. 创建实例（启动可运行的团队副本）
```

---

## 第一步：新建提示词

提示词（Prompt）是智能体的系统指令模板，可在多个智能体间复用。

**接口**：`POST /api/prompts`

```bash
curl -X POST http://localhost:8000/api/prompts \
  -H "Content-Type: application/json" \
  -d '{
    "prompt_key": "前端工程师提示词",
    "name": "前端工程师提示词",
    "content_md": "## 角色\n你是一个专业的前端工程师，负责 UI 设计和实现。\n\n## 能力\n- 熟练掌握 React/Vue 等现代前端框架\n- 注重用户体验和界面美观\n- 能够独立完成页面开发\n\n## 工作方式\n接到任务后先分析需求，再逐步实现代码。",
    "description": "前端工程师的系统提示词模板",
    "tags": ["前端", "UI"],
    "enabled": true
  }'
```

返回示例：

```json
{
  "ok": true,
  "prompt_key": "前端工程师提示词"
}
```

> **提示**：建议为团队中每个角色各建一个提示词，便于维护。

---

## 第二步：新建 MCP（可选）

MCP（Model Context Protocol）服务可扩展智能体的工具能力，例如接入文件系统、数据库、搜索等。

**接口**：`POST /api/capabilities/mcp`

```bash
# stdio 模式（本地进程）
curl -X POST http://localhost:8000/api/capabilities/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "name": "filesystem",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"],
    "description": "本地文件系统访问",
    "transport": "stdio",
    "cwd": "/Users/xiaoyemiao/Desktop/meowone",
    "auth_type": "none"
  }'

# SSE 模式（远程服务）
curl -X POST http://localhost:8000/api/capabilities/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "name": "remote-mcp",
    "description": "远程 MCP 服务",
    "transport": "sse",
    "url": "https://your-mcp-server.com/mcp",
    "auth_type": "bearer",
    "auth_token": "your-token-here"
  }'
```

返回示例：

```json
{
  "ok": true,
  "name": "filesystem"
}
```

> **说明**：`transport` 支持 `stdio`（本地进程）、`sse`、`streamable-http`（远程服务）三种模式。

## 第三步：新建 Skill（可选）

Skill 是智能体技能库，支持自然语言触发，适合让智能体在特定场景下执行预设工作流。

**接口**：`POST /api/capabilities/skills`

```bash
curl -X POST http://localhost:8000/api/capabilities/skills \
  -H "Content-Type: application/json" \
  -d '{
    "name": "代码审查",
    "description": "对代码进行质量审查并给出改进建议",
    "body": "# 代码审查技能\n\n当用户要求审查代码时，执行以下步骤：\n1. 阅读代码，理解逻辑\n2. 检查潜在 bug 和安全问题\n3. 评估代码风格和可维护性\n4. 给出具体改进建议",
    "trigger_keywords": ["审查代码", "review", "代码审查"],
    "category": "development",
    "examples": ["帮我审查这段代码"],
    "version": "1.0.0"
  }'
```

返回示例：

```json
{
  "ok": true,
  "name": "代码审查"
}
```

## 第四步：创建智能体

智能体（Agent）是团队的核心成员，可以是**内部智能体**（平台内运行）或**外部智能体**（通过 A2A 协议远程调用）。

### 4.1 创建内部智能体

**接口**：`POST /api/agents/internal`

```bash
curl -X POST http://localhost:8000/api/agents/internal \
  -H "Content-Type: application/json" \
  -d '{
    "name": "前端工程师",
    "description": "负责前端开发任务，包括 UI 设计和实现",
    "system_prompt": "## 角色\n你是一个专业的前端工程师，负责 UI 设计和实现。\n\n## 能力\n- 熟练掌握 React/Vue 等现代前端框架\n- 注重用户体验和界面美观\n\n## 工作方式\n接到任务后先分析需求，再逐步实现代码。",
    "prompt_key": "前端工程师提示词",
    "model_name": "qwen-plus",
    "scheduler_mode": "direct",
    "loop_mode": "react",
    "mcp_servers": ["filesystem"],
    "agent_skills": ["代码审查"],
    "allow_tools": [],
    "deny_tools": [],
    "max_rounds": 20,
    "timeout_seconds": 300
  }'
```

> **参数说明**：
> - `scheduler_mode`：调度模式，`direct`（直接调度）、`capability_match`（能力匹配）等
> - `loop_mode`：执行循环模式，`react`（ReAct）或 `plan_exec`（计划执行）
> - `mcp_servers`：引用第二步创建的 MCP 服务名称列表
> - `agent_skills`：引用第三步创建的 Skill 名称列表
> - `prompt_key`：引用第一步创建的提示词 key

重复此步骤，为团队中每个角色各创建一个智能体。例如：

| 角色 | name 示例 |
|------|-----------|
| 前端工程师 | `前端工程师` |
| 后端工程师 | `后端工程师` |
| 产品经理 | `产品经理` |
| 测试工程师 | `测试工程师` |
| 技术领导 | `技术领导` |

### 4.2 查询刚创建的智能体 ID

创建后需要记录智能体的 UUID，用于后续步骤。

```bash
# 列出所有内部智能体，找到刚创建的智能体
curl -s http://localhost:8000/api/agents?agent_type=internal | python3 -m json.tool

# 按名称查询
curl -s http://localhost:8000/api/agents/internal/前端工程师 | python3 -m json.tool
```

返回示例（`id` 字段即智能体 ID）：

```json
{
  "id": "ecc37ff2-bf22-40aa-a539-05ed67bb1fee",
  "name": "前端工程师",
  "description": "负责前端开发任务，包括 UI 设计和实现",
  "agent_type": "internal",
  "enabled": true,
  ...
}
```

记录下每个智能体的 `id`，在下一步创建镜像时会用到。

## 第五步：创建镜像

镜像（Image）将多个智能体打包为一个可复用的组合单元，同时关联调度策略。

**接口**：`POST /api/v3/images`

```bash
curl -X POST http://localhost:8000/api/v3/images \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的团队",
    "description": "包含前端、后端、产品经理、测试、技术领导的开发团队",
    "agent_ids": [
      "ecc37ff2-bf22-40aa-a539-05ed67bb1fee",
      "be8d8666-da31-4c0c-9171-aa963108064e",
      "d1974a14-fa4c-471b-a9ec-c10f66555712",
      "cb12b7de-35df-4498-a617-146ea280111a",
      "45069004-1ba3-481c-8fa0-0fb1026f9ecc"
    ],
    "loop_id": null,
    "strategy_id": null,
    "environment_id": null,
    "metadata": {}
  }'
```

返回示例：

```json
{
  "ok": true,
  "image": {
    "id": "9d4892fd-e4a8-4f01-b752-69fb9c919a4b",
    "name": "我的团队",
    "description": "包含前端、后端、产品经理、测试、技术领导的开发团队",
    "agent_ids_json": [...],
    ...
  }
}
```

> **说明**：`agent_ids` 填入第四步获取的智能体 UUID 列表。`strategy_id` 可留空，调度策略在第六步单独配置。

## 第六步：创建调度配置

调度配置（Strategy Config）决定如何将任务分发给团队成员。平台内置以下策略：

| 策略 | 说明 |
|------|------|
| `direct` | 直接调度：调用单个智能体处理任务 |
| `team_dispatch` | 团队调度：将任务分发给团队成员 |
| `capability_match` | 能力匹配：根据任务需求匹配最佳智能体 |

**6.1 查询可用策略**

```bash
curl -s http://localhost:8000/api/v3/strategies | python3 -m json.tool
```

返回示例：

```json
{
  "count": 3,
  "strategies": [
    {
      "id": "e6be933e-7cc9-4aab-a0ff-ec72d1295284",
      "name": "direct",
      "description": "直接调度：直接调用单个智能体处理任务"
    },
    {
      "id": "5f98a09f-1dee-4391-90dd-9aafb2190f30",
      "name": "team_dispatch",
      "description": "团队调度：将任务分发给团队成员智能体"
    },
    {
      "id": "6c1faf8e-8686-4782-ad5e-44ccb1c41735",
      "name": "capability_match",
      "description": "能力匹配：根据任务需求匹配最佳智能体"
    }
  ]
}
```

**6.2 创建策略配置**

**使用 capability_match（能力匹配）**：

`config` 可留空，策略会自动从镜像的 `agent_ids_json` 构建候选池。如需指定匹配的模型，可传入 `match_model`：

```bash
curl -X POST http://localhost:8000/api/v3/strategy-configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的团队-能力匹配配置",
    "description": "通过能力匹配从团队中选择最合适的成员执行任务",
    "image_id": "9d4892fd-e4a8-4f01-b752-69fb9c919a4b",
    "strategy_id": "6c1faf8e-8686-4782-ad5e-44ccb1c41735",
    "config": {
      "match_model": "qwen-plus"
    }
  }'
```

**config 字段说明（capability_match）**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `match_model` | string | 否 | 指定用于语义匹配的模型，不填则使用 instance 的 model |

**使用 team_dispatch（团队调度）**：

`team_id` **必填**，其余字段有默认值：

```bash
curl -X POST http://localhost:8000/api/v3/strategy-configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的团队-团队调度配置",
    "description": "将任务分发给团队成员并行或串行处理",
    "image_id": "9d4892fd-e4a8-4f01-b752-69fb9c919a4b",
    "strategy_id": "5f98a09f-1dee-4391-90dd-9aafb2190f30",
    "config": {
      "team_id": "我的团队",
      "parallel": true,
      "max_members": 5
    }
  }'
```

**config 字段说明（team_dispatch）**：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `team_id` | string | **是** | - | 团队标识符（业务语义，用于日志和识别） |
| `decompose_prompt` | string | 否 | 系统默认分解提示词 | 自定义任务分解提示词 |
| `parallel` | bool | 否 | `true` | `true`：成员并行执行；`false`：串行执行 |
| `max_members` | int | 否 | `5` | 最多使用几个成员，超出部分忽略 |

返回示例：

```json
{
  "ok": true,
  "config": {
    "id": "4f8244e9-b464-4005-91a1-709ea8c49af2",
    "name": "我的团队-能力匹配配置",
    "image_id": "9d4892fd-e4a8-4f01-b752-69fb9c919a4b",
    "strategy_id": "6c1faf8e-8686-4782-ad5e-44ccb1c41735",
    ...
  }
}
```

> **注意**：`strategy_id` 填入上一步查到的策略 UUID（而非策略名称字符串）。

## 第七步：创建执行环境（可选）

执行环境（Environment）定义智能体运行时的资源限制、工具权限和超时策略。

**接口**：`POST /api/v3/environments`

```bash
curl -X POST http://localhost:8000/api/v3/environments \
  -H "Content-Type: application/json" \
  -d '{
    "name": "生产环境",
    "description": "正式环境的沙箱配置",
    "sandbox_type": "native",
    "sandbox_config": {},
    "resource_limits": {
      "max_memory_mb": 512,
      "max_cpu_percent": 80
    },
    "allowed_tools": ["filesystem", "web_search"],
    "denied_tools": ["rm", "format_disk"],
    "max_rounds": 20,
    "timeout_seconds": 300
  }'
```

返回示例：

```json
{
  "ok": true,
  "environment": {
    "id": "...",
    "name": "生产环境",
    ...
  }
}
```

> **说明**：如不创建执行环境，系统默认使用 `native` 沙箱，不限制工具权限。

## 向团队发送任务

拿到 `instance_id` 后，即可通过聊天接口向团队发任务。

**接口**：`POST /api/sessions/{session_id}/chat`

```bash
# 1. 先创建一个会话
SESSION_RESP=$(curl -s -X POST http://localhost:8000/api/sessions \
  -H "Content-Type: application/json" \
  -d '{"title": "测试会话"}')
echo $SESSION_RESP
# 提取 session_id，例如："id": "abc123"

# 2. 向团队发任务（使用 instance_id 路由到团队）
curl -X POST http://localhost:8000/api/sessions/abc123/chat \
  -H "Content-Type: application/json" \
  -N -s -H "Accept: text/event-stream" \
  -d '{
    "content": "请帮我开发一个用户登录页面，包含用户名密码输入和登录按钮",
    "instance_id": "6785ab85-3872-4cd2-9eef-5034345e98bc"
  }'
```

> **SSE 响应**：聊天接口返回 SSE 流式事件，包含 `thinking`、`delta`（文本增量）、`tool_call`、`tool_result`、`error`、`done` 等事件类型。

## 完整流程示例汇总

以下是一个最小化团队的完整创建流程（所有参数为示例值，请替换为实际 ID）：

```bash
# === 1. 创建提示词 ===
curl -X POST http://localhost:8000/api/prompts \
  -H "Content-Type: application/json" \
  -d '{"prompt_key": "fe-prompt", "name": "前端提示词", "content_md": "...", "enabled": true}'

# === 2. 创建智能体（重复创建多个）===
# 智能体1：前端工程师
FE_RESP=$(curl -s -X POST http://localhost:8000/api/agents/internal \
  -H "Content-Type: application/json" \
  -d '{"name": "前端工程师", "description": "负责前端开发", "prompt_key": "fe-prompt", "loop_mode": "react"}')
FE_ID=$(echo $FE_RESP | python3 -c "import sys,json; print(json.load(sys.stdin).get('name',''))")
# 注意：上方命令仅示例。实际请从列表接口获取智能体 UUID

# === 3. 列出智能体获取 UUID ===
curl -s http://localhost:8000/api/agents?agent_type=internal

# === 4. 创建镜像 ===
curl -X POST http://localhost:8000/api/v3/images \
  -H "Content-Type: application/json" \
  -d '{"name": "开发团队", "agent_ids": ["<ID1>", "<ID2>", "<ID3>"]}'

# === 5. 创建调度配置 ===
curl -X POST http://localhost:8000/api/v3/strategy-configs \
  -H "Content-Type: application/json" \
  -d '{"name": "开发团队-能力匹配", "image_id": "<IMAGE_ID>", "strategy_id": "<CAPABILITY_MATCH_UUID>", "config": {}}'

# === 6. 创建实例 ===
curl -X POST http://localhost:8000/api/v3/instances \
  -H "Content-Type: application/json" \
  -d '{"name": "开发团队-实例", "image_id": "<IMAGE_ID>", "strategy_config_id": "<CONFIG_ID>"}'

# === 7. 开始对话 ===
curl -X POST http://localhost:8000/api/sessions/<SESSION_ID>/chat \
  -H "Content-Type: application/json" \
  -N -s -H "Accept: text/event-stream" \
  -d '{"content": "帮我开发一个用户登录页面", "instance_id": "<INSTANCE_ID>"}'
```

## 常见问题

**Q: 提示词、智能体、镜像之间的关系是什么？**

提示词是智能体的系统指令模板（可复用），智能体引用提示词，镜像将多个智能体打包。类似于：模板 → 实例 → 组合包。

**Q: strategy_id 应该填策略名称还是 UUID？**

在**策略配置**（`/api/v3/strategy-configs`）中，`strategy_id` 填 **UUID**（如 `6c1faf8e-8686-4782-ad5e-44ccb1c41735`）。调度引擎内部会通过 UUID 查找策略名称。

**Q: 如何切换调度策略？**

创建新的策略配置（第六步）指向同一镜像但不同的 `strategy_id`，然后在实例中更新 `strategy_config_id` 即可：

```bash
curl -X PUT http://localhost:8000/api/v3/instances/<INSTANCE_ID> \
  -H "Content-Type: application/json" \
  -d '{"strategy_config_id": "<新配置ID>"}'
```

**Q: 团队成员需要手动分配任务吗？**

不需要。调度策略（`capability_match` 或 `team_dispatch`）会自动根据任务内容决定分发对象。`direct` 策略则直接调用指定智能体。
