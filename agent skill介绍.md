# Agent Skill 介绍

## 什么是 Agent Skill？

Agent Skill（智能体技能）是由 Anthropic 提出并推广的一种模块化能力扩展标准，旨在让 AI Agent（智能体）能够像调用"技能包"一样，按需动态加载并执行特定领域的任务。

Agent Skill 本质上是一个**包含指令、元数据和可选资源（如脚本、模板、参考文档）的文件夹**。它将专业知识、工作流规范（SOP）和操作工具固化为可复用的资产，使 AI 从单纯的"对话者"转变为能够自主执行任务的"执行者"。

---

## 核心工作原理：渐进式披露（Progressive Disclosure）

为了避免上下文窗口（Context Window）被冗长的提示词填满，Agent Skill 采用了分层加载机制：

| 层级 | 内容 | 加载时机 |
|------|------|----------|
| **L1 元数据** | 仅加载 `SKILL.md` 中的 YAML 元数据（名称与描述） | 始终加载，让 AI 识别何时需要该技能 |
| **L2 核心指令** | `SKILL.md` 的详细操作指南 | 当 AI 判定任务匹配时加载 |
| **L3+ 资源与脚本** | 外部脚本（如 Python、Bash）或参考文档 | 仅在执行具体步骤时按需加载 |

这种机制可以大幅降低 Token 消耗，避免指令漂移。

---

## 为什么需要 Agent Skill？

### 1. 解决重复性低效
无需在每次对话中重复输入复杂的业务流程，将 SOP 固化为 Skill 即可一键调用。

### 2. 优化上下文效率
通过渐进式披露机制，大幅降低 Token 消耗，保持对话简洁高效。

### 3. 知识复用与共享
Skill 可以跨项目、跨团队共享，支持版本管理，将个人的隐性经验转化为团队的显性资产。

### 4. 增强执行力
支持封装可执行脚本，使 AI 能够直接操作文件、调用 API 或执行系统命令，而不仅仅是提供建议。

## Agent Skill vs 其他概念

### 与 Prompt 的区别

| 特性 | Prompt | Agent Skill |
|------|--------|-------------|
| 形态 | 静态的指令 | 模块化、可复用的能力包 |
| 触发 | 每次手动输入 | 可自动触发 |
| 脚本支持 | 无 | 支持封装可执行脚本 |
| 复用性 | 低 | 高 |

### 与 MCP（模型上下文协议）的区别

| 特性 | MCP | Agent Skill |
|------|-----|-------------|
| 定位 | 工具箱 | 工作手册 |
| 核心 | AI 如何连接外部数据源和工具 | 如何完成特定任务的方法论和执行逻辑 |
| 作用 | 连接"能做什么" | 指导"该怎么做" |

简单来说：**MCP 告诉你有什么工具可用，Agent Skill 告诉你怎么用好这些工具。**

## Agent Skill 在 Cursor 中的应用

Cursor Agent Skills 是一种用于为 AI Agent 扩展专门能力的开放标准，它将特定领域的知识、工作流和工具打包成可复用的模块，使 Agent 能够根据上下文自动调用相关技能。

### 存储位置

- **个人全局技能**：`~/.cursor/skills/`
- **项目级技能**：`<项目根目录>/.cursor/skills/`

> 注意：不要修改 `~/.cursor/skills-cursor/` 目录，这是内置技能区。

### 技能文件结构（SKILL.md）

每个技能必须包含一个 `SKILL.md` 文件，采用 YAML Frontmatter 格式：

```markdown
---
name: skill-name
description: 描述技能功能及触发场景
---

# 技能名称

## Instructions
- 详细的步骤指南
- 领域特定规范
```

**字段说明：**
- **name**：唯一标识，仅限小写字母、数字和连字符
- **description**：AI 判断何时调用该技能的关键，需清晰描述功能与场景

### 调用方式

1. **自动调用**：Agent 根据任务上下文自动匹配
2. **手动调用**：在对话框中使用 `/` 加技能名称，或使用 `@` 引用技能

---

## 典型目录结构

```
skill-name/
├── SKILL.md           # 核心文件，包含元数据和指令
├── scripts/           # 存放可执行脚本（如 Python、Bash）
├── references/        # 存放参考文档
└── assets/            # 存放模板或素材
```

## 最佳实践

### 1. MVP 先行
先梳理工作流，通过 `skill-creator` 等工具快速构建原型，验证效果后再逐步完善。

### 2. 渐进式加载
利用 Skills 的按需加载特性，保持上下文简洁，避免 Token 浪费。

### 3. 版本管理
将技能文件纳入 Git 管理，方便团队协作与版本追踪。

### 4. 清晰的描述
description 字段要写得清晰准确，让 AI 能够准确判断何时触发该技能。

### 5. 调试技巧
若技能未生效，可使用 `/reload-skills` 命令刷新缓存，并检查 `SKILL.md` 的 YAML 格式是否正确。

---

## 总结

Agent Skill 目前已成为 AI 工程领域的重要标准，广泛应用于 Claude Code、Cursor、VS Code 等开发工具中。它是构建可落地、专业化 AI Agent 的关键基础设施，能够帮助：

- 将重复性工作流程自动化
- 团队内部共享专业知识和最佳实践
- 提升 AI Agent 的专业能力和执行效率
- 降低使用成本（通过渐进式加载减少 Token 消耗）

通过合理使用 Agent Skill，可以让 AI Agent 从一个通用助手，成长为一个具备专业领域知识的得力助手。
