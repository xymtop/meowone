# MeowOne AI 操作系统 — 用户手册

## 一、系统概述

### 1.1 什么是 MeowOne？

MeowOne 是一个**以对话为核心入口的 AI 操作系统**，用户通过自然语言描述需求，系统理解意图、调度工具与智能体，并通过流式响应返回思考过程、正文与卡片式 UI。

**核心特点：**
- 以对话为核心的交互方式
- 多智能体协作执行任务
- 工具并行执行提高效率
- 流式响应实时反馈
- 卡片式 UI 展示丰富信息

### 1.2 系统架构

MeowOne 采用分层架构设计，包括：
- **前端层**：Next.js 应用，提供用户界面
- **后端层**：FastAPI 应用，处理 API 请求和智能体编排
- **服务层**：LLM API、SQLite 数据库、MCP 服务、A2A 子智能体

## 二、快速开始

### 2.1 环境准备

**后端配置：**
1. 复制 `backend/.env.example` 为 `backend/.env`
2. 配置 LLM API 相关参数：
   ```env
   LLM_API_KEY=sk-your-api-key
   LLM_BASE_URL=https://api.openai.com/v1
   LLM_MODEL=gpt-4o
   ```

**前端配置：**
1. 复制根目录 `.env.example` 为 `admin_web/.env.local`
2. 配置 API 地址：
   ```env
   NEXT_PUBLIC_API_URL=http://localhost:8000
   ```

### 2.2 启动服务

**启动后端：**
```bash
cd backend
pip3 install -r requirements.txt
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**启动前端：**
```bash
cd admin_web
npm install
npm run dev
```

**访问地址：**
- 后端：http://localhost:8000
- 前端：http://localhost:3000

### 2.3 首次使用

1. 打开前端页面 http://localhost:3000
2. 点击「创建会话」按钮
3. 在对话框中输入您的需求，例如：「帮我写一个 Python 函数，计算斐波那契数列」
4. 系统会自动处理并返回结果

## 三、主要功能使用指南

### 3.1 会话管理

**创建会话：**
1. 在前端页面点击「创建会话」按钮
2. 输入会话名称（可选）
3. 选择智能体类型（可选）
4. 点击「确定」创建会话

**管理会话：**
- **查看会话列表**：在左侧边栏查看所有会话
- **切换会话**：点击会话名称即可切换
- **删除会话**：点击会话右侧的删除按钮
- **重命名会话**：点击会话名称进行编辑

### 3.2 智能体管理

**创建内部智能体：**
1. 导航到「智能体」页面
2. 点击「创建智能体」按钮
3. 选择「内部智能体」
4. 填写智能体信息：
   - 名称：智能体的唯一标识
   - 描述：智能体的功能描述
   - 系统提示词：定义智能体的角色和行为
   - 能力：选择智能体可以使用的工具
   - 循环类型：选择智能体的思考方式
5. 点击「创建」按钮

**创建外部智能体：**
1. 导航到「智能体」页面
2. 点击「创建智能体」按钮
3. 选择「外部智能体」
4. 填写智能体信息：
   - 名称：智能体的唯一标识
   - 描述：智能体的功能描述
   - 基础 URL：远程智能体的地址
   - 协议：选择 A2A 协议
5. 点击「创建」按钮

### 3.3 工具使用

**内置工具：**
- **bash**：执行命令行操作
- **read_workspace_file**：读取工作区文件
- **write_workspace_file**：写入工作区文件
- **list_workspace_dir**：列出目录内容
- **call_mcp_tool**：调用 MCP 工具
- **subagent_tool**：调用子智能体

**使用工具示例：**
```
用户：帮我查看当前目录下的文件

系统：当前目录包含以下文件：
- README.md
- backend/
- admin_web/
- agents/
- docs/
```

### 3.4 技能管理

**查看技能：**
1. 导航到「技能」页面
2. 查看系统中已有的技能

**创建技能：**
1. 导航到「技能」页面
2. 点击「创建技能」按钮
3. 填写技能信息：
   - 名称：技能的唯一标识
   - 描述：技能的功能描述
   - 内容：技能的具体实现
4. 点击「创建」按钮

### 3.5 工作流管理

**创建工作流：**
1. 导航到「工作流」页面
2. 点击「创建工作流」按钮
3. 填写工作流信息：
   - 名称：工作流的唯一标识
   - 描述：工作流的功能描述
   - 节点：添加工作流节点
   - 连接：设置节点之间的连接关系
4. 点击「创建」按钮

**执行工作流：**
1. 在工作流列表中找到要执行的工作流
2. 点击「执行」按钮
3. 填写工作流参数
4. 点击「开始执行」按钮

## 四、智能体类型与使用场景

### 4.1 循环类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| `react` | 标准 ReAct 循环（思考→行动→观察） | 通用任务，如问答、简单工具调用 |
| `plan_exec` | 计划-执行分离 | 需要先规划后执行的复杂任务 |
| `critic` | 执行+评审双循环 | 高质量输出要求，如代码审查、文档编写 |
| `hierarchical` | 层级式循环 | 复杂多步骤任务，需要分工协作 |
| `multi_agent_debate` | 多智能体辩论 | 需要多角度分析的任务，如决策制定 |

### 4.2 智能体示例

**代码助手：**
- 循环类型：`react`
- 能力：`bash`、`read_workspace_file`、`write_workspace_file`
- 适用场景：代码编写、重构、审查

**数据分析师：**
- 循环类型：`plan_exec`
- 能力：`bash`、`read_workspace_file`、`call_mcp_tool`
- 适用场景：数据分析、报告生成

**文档助手：**
- 循环类型：`critic`
- 能力：`read_workspace_file`、`write_workspace_file`
- 适用场景：文档撰写、摘要、改写

**项目管理助手：**
- 循环类型：`hierarchical`
- 能力：`bash`、`read_workspace_file`、`write_workspace_file`、`subagent_tool`
- 适用场景：项目规划、任务分配、进度跟踪

## 五、常见场景示例

### 5.1 代码编写

**任务：** 帮我写一个 Python 函数，计算斐波那契数列

**操作步骤：**
1. 创建一个新会话
2. 选择「代码助手」智能体
3. 输入任务描述：「帮我写一个 Python 函数，计算斐波那契数列」
4. 系统会自动生成代码并返回结果

**预期结果：**
```python
def fibonacci(n):
    """计算斐波那契数列的第 n 项"""
    if n <= 0:
        return 0
    elif n == 1:
        return 1
    else:
        return fibonacci(n-1) + fibonacci(n-2)

# 测试函数
print(fibonacci(10))  # 输出: 55
```

## 六、故障排除

### 6.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 后端启动失败 | LLM API 配置错误 | 检查 `backend/.env` 中的 LLM 相关配置 |
| 前端无法连接后端 | API 地址配置错误 | 检查 `admin_web/.env.local` 中的 `NEXT_PUBLIC_API_URL` |
| 工具执行失败 | 工具参数错误或权限问题 | 检查工具参数格式，确保工作区路径正确 |
| 智能体循环超时 | 工具执行时间过长 | 调整 `LOOP_TIMEOUT_SECONDS` 和 `TOOL_TIMEOUT_SECONDS` 配置 |
| 远程智能体调用失败 | 网络连接问题 | 检查远程智能体地址是否正确，网络是否可达 |

### 6.2 日志查看

- 控制台输出：运行后端时的终端输出

2. **合理设置超时**：根据任务复杂度设置适当的超时时间
3. **使用流式输出**：开启流式响应，提升用户体验
4. **监控资源使用**：关注 token 消耗和执行时间

## 八、高级功能

### 8.1 A2A 子智能体

**创建 A2A 子智能体：**
1. 进入 `agents/` 目录
2. 选择一个子智能体模板（如 `code-writer`）
3. 配置 `.env` 文件
4. 启动子智能体：
   ```bash
   cd agents/code-writer
   pip install -e .
   python __main__.py
   ```

**配置 A2A 子智能体：**
1. 编辑 `.meowone/agents.yaml` 文件
2. 添加子智能体配置：
   ```yaml
   tools:
     - tool_name: code_writer
       description: 代码编写、重构、审查
       base_url: http://127.0.0.1:8001
       enabled: true
   ```

### 8.2 MCP 服务集成

**添加 MCP 服务：**
1. 导航到「能力」页面
2. 点击「添加 MCP 服务」按钮
3. 填写 MCP 服务信息：
   - 名称：服务的唯一标识
   - 命令：启动 MCP 服务的命令
   - 参数：命令参数
4. 点击「添加」按钮

**使用 MCP 服务：**
1. 创建智能体时选择 MCP 服务
2. 在对话中使用 MCP 工具

### 8.3 自定义工具

**创建自定义工具：**
1. 在 `backend/app/capability/tools/` 目录创建新工具文件
2. 实现工具逻辑
3. 在 `backend/app/capability/__init__.py` 中注册工具

**使用自定义工具：**
1. 创建智能体时选择自定义工具
2. 在对话中使用自定义工具

## 九、API 参考

### 9.1 核心 API

| 路径 | 方法 | 功能 |
|------|------|------|
| `/health` | GET | 健康检查 |
| `/api/sessions` | GET | 获取会话列表 |
| `/api/sessions` | POST | 创建会话 |
| `/api/sessions/{id}/messages` | GET | 获取消息历史 |
| `/api/sessions/{session_id}/chat` | POST | 发送消息，SSE 流式返回 |
| `/api/agents` | GET | 获取智能体列表 |
| `/api/agents/internal` | POST | 创建内部智能体 |
| `/api/agents/external` | POST | 创建外部智能体 |
| `/api/capabilities` | GET | 获取能力列表 |
| `/api/models` | GET | 获取模型列表 |
| `/v1/chat/completions` | POST | OpenAI 兼容接口 |

### 9.2 流式事件类型

| 事件 | 说明 | 数据结构 |
|------|------|----------|
| `thinking` | 智能体思考中 | `{"step": 1, "description": "正在思考..."}` |
| `delta` | 文本增量输出 | `{"message_id": "uuid", "content": "文本内容", "done": false}` |
| `tool_call` | 工具调用 | `{"tool_call_id": "uuid", "capability_name": "tool_name", "params": {}}` |
| `tool_result` | 工具执行结果 | `{"tool_call_id": "uuid", "capability_name": "tool_name", "result": {}, "success": true}` |
| `card` | 卡片输出 | `{"message_id": "uuid", "card": {"type": "info", "title": "标题", "content": "内容"}}` |
| `error` | 错误信息 | `{"code": "ERROR_CODE", "message": "错误信息"}` |
| `done` | 任务完成 | `{"message_id": "uuid", "loop_rounds": 3, "total_duration": 1234}` |
