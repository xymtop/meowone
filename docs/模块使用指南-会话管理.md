# 会话管理模块使用指南

## 一、功能概述

会话管理模块是 MeowOne AI 操作系统的核心功能之一，负责管理用户与智能体之间的对话会话。每个会话代表一个独立的对话上下文，系统会保存会话历史，支持上下文连续对话。

## 二、核心概念

### 2.1 会话（Session）

会话是用户与智能体交互的基本单位，包含：
- 会话ID：唯一标识符
- 会话名称：用户自定义的会话名称
- 创建时间：会话创建的时间戳
- 消息历史：会话中的所有消息记录

### 2.2 消息（Message）

消息是会话中的基本交互单元，包含：
- 角色（role）：用户（user）、助手（assistant）、工具（tool）
- 内容（content）：消息的文本内容
- 时间戳：消息发送的时间

## 三、主要功能

### 3.1 创建会话

**功能说明**：创建一个新的会话，开始与智能体的交互。

**操作步骤**：
1. 点击左侧边栏的「创建会话」按钮
2. 在弹出的对话框中输入会话名称（可选）
3. 选择智能体类型（可选，如果不选择则使用默认智能体）
4. 点击「确定」按钮创建会话

**API 接口**：
```
POST /api/sessions
Content-Type: application/json

{
  "name": "我的新会话",
  "agent_id": "default-agent"
}
```

### 3.2 查看会话列表

**功能说明**：查看所有已创建的会话。

**操作步骤**：
- 在左侧边栏可以看到所有会话的列表
- 会话按创建时间倒序排列，最新的会话在最上面

**API 接口**：
```
GET /api/sessions
```

### 3.3 切换会话

**功能说明**：在不同的会话之间切换。

**操作步骤**：
1. 在左侧边栏找到要切换的会话
2. 点击会话名称即可切换到该会话
3. 系统会加载该会话的所有历史消息

### 3.4 重命名会话

**功能说明**：修改会话的名称，便于识别和管理。

**操作步骤**：
1. 在左侧边栏找到要重命名的会话
2. 点击会话名称右侧的「更多选项」按钮（三个点）
3. 选择「重命名」选项
4. 输入新的会话名称
5. 点击「确定」按钮保存

**API 接口**：
```
PATCH /api/sessions/{id}
Content-Type: application/json

{
  "name": "新的会话名称"
}
```

### 3.5 删除会话

**功能说明**：删除不再需要的会话，释放存储空间。

**操作步骤**：
1. 在左侧边栏找到要删除的会话
2. 点击会话名称右侧的「更多选项」按钮（三个点）
3. 选择「删除」选项
4. 在确认对话框中点击「确定」按钮

**注意事项**：
- 删除会话会同时删除该会话中的所有消息历史
- 删除操作不可恢复，请谨慎操作

**API 接口**：
```
DELETE /api/sessions/{id}
```

### 3.6 查看会话详情

**功能说明**：查看会话的详细信息和消息历史。

**操作步骤**：
1. 切换到要查看的会话
2. 在主界面可以看到该会话的所有消息历史
3. 可以向上滚动查看更早的消息

**API 接口**：
```
GET /api/sessions/{id}
GET /api/sessions/{id}/messages
```

## 四、会话消息管理

### 4.1 发送消息

**功能说明**：向智能体发送消息，开始对话。

**操作步骤**：
1. 在会话页面底部的输入框中输入消息内容
2. 可以选择添加附件（图片、文档等）
3. 点击「发送」按钮或按 Enter 键发送消息
4. 系统会显示思考过程和回复内容

**API 接口**：
```
POST /api/sessions/{session_id}/chat
Content-Type: application/json

{
  "messages": [
    {"role": "user", "content": "你好，请帮我写一个Python函数"}
  ],
  "stream": true
}
```

### 4.2 查看消息历史

**功能说明**：查看会话中的所有消息记录。

**消息类型**：
- 用户消息：用户发送的消息
- 助手消息：智能体回复的消息
- 工具调用：智能体调用工具的记录
- 工具结果：工具执行的结果
- 思考过程：智能体的思考状态

### 4.3 复制消息

**功能说明**：复制消息内容到剪贴板。

**操作步骤**：
1. 找到要复制的消息
2. 点击消息右侧的「复制」按钮
3. 消息内容会自动复制到剪贴板

### 4.4 重新生成回复

**功能说明**：让智能体重新生成上一条回复。

**操作步骤**：
1. 找到智能体的最后一条回复
2. 点击消息右侧的「重新生成」按钮
3. 系统会重新思考并生成新的回复

## 五、流式响应事件

当发送消息时，系统会通过 SSE（Server-Sent Events）流式返回以下事件：

| 事件类型 | 说明 | 数据示例 |
|---------|------|---------|
| `thinking` | 智能体思考中 | `{"step": 1, "description": "正在分析问题..."}` |
| `delta` | 文本增量输出 | `{"content": "好的，我来", "done": false}` |
| `tool_call` | 工具调用 | `{"tool": "read_file", "params": {"path": "test.py"}}` |
| `tool_result` | 工具执行结果 | `{"tool": "read_file", "result": "文件内容"}` |
| `card` | 卡片输出 | `{"type": "info", "title": "结果", "content": "..."}` |
| `error` | 错误信息 | `{"code": "ERROR", "message": "出错了"}` |
| `done` | 任务完成 | `{"loop_rounds": 3, "total_duration": 1234}` |

## 六、最佳实践

### 6.1 会话组织建议

1. **按主题分组**：每个会话专注于一个主题或任务
2. **合理命名**：使用清晰的会话名称，便于后续查找
3. **及时清理**：定期删除不再需要的会话

### 6.2 对话技巧

1. **具体明确**：提供详细的任务描述
2. **提供上下文**：如果需要参考文件，明确指出
3. **逐步细化**：对于复杂任务，可以分步骤描述
4. **及时反馈**：如果回复不符合预期，及时指出并提供反馈

### 6.3 性能优化

1. **控制会话长度**：过长的会话会影响响应速度
2. **合理拆分任务**：对于特别复杂的任务，可以拆分为多个会话
3. **及时清理历史**：定期清理不需要的历史会话

## 七、常见问题

### Q1: 会话被删除了可以恢复吗？

A: 目前删除会话是不可逆的操作，删除后无法恢复。请在删除前确认。

### Q2: 可以导出会话历史吗？

A: 可以通过 API 获取会话的所有消息，然后自行导出。

### Q3: 会话有长度限制吗？

A: 系统会自动管理会话上下文，确保不超过 LLM 的最大上下文长度限制。

### Q4: 可以同时打开多个会话吗？

A: 可以在浏览器中打开多个标签页，每个标签页显示不同的会话。
