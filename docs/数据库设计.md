# MeowOne 数据库设计（v2）

本文档描述当前后端 SQLite 数据库的实体模型、关系约束、索引与设计原则，用于研发、联调、排障与后续迁移。

---

## 1. 设计目标

- 以会话与消息为主链路，支撑对话式 AI 操作系统。
- 以数据库作为配置主数据源，统一管理 MCP、Skill、Model、Agent、Menu 等平台能力。
- 在保证演进效率的前提下，补齐关键关系约束，减少孤儿数据与脏数据风险。

---

## 2. 数据库与引擎

- 引擎：SQLite（`aiosqlite`）
- 数据库文件：`DATABASE_PATH`（默认 `meowone.db`）
- 连接策略：
  - 每次连接开启 `PRAGMA foreign_keys = ON`
  - 初始化时执行全量建表脚本（`CREATE TABLE IF NOT EXISTS ...`）

---

## 3. ER 概览

- `users` 1 - N `sessions`
- `sessions` 1 - N `messages`
- `sessions` 1 - N `loop_logs`
- `sessions` 1 - N `channel_sessions`
- `menus` 自关联（`parent_key -> menu_key`）

配置/能力相关表（逻辑上独立）：

- `mcp_servers`
- `skills`
- `llm_models`
- `prompts`
- `agents`
- `scheduled_tasks`
- `menus`

---

## 4. 表结构说明

## 4.1 用户与会话域

### `users`
- `id` TEXT PK
- `username` TEXT UNIQUE NOT NULL
- `created_at`

### `sessions`
- `id` TEXT PK
- `user_id` TEXT NOT NULL
- `title`
- `summary`
- `created_at`
- `updated_at`

### `channel_sessions`
- `id` TEXT PK
- `channel_id` TEXT NOT NULL
- `external_thread_id` TEXT NOT NULL
- `session_id` TEXT NOT NULL
- `created_at`
- `updated_at`
- 约束：
  - `UNIQUE(channel_id, external_thread_id)`
  - `FOREIGN KEY(session_id) REFERENCES sessions(id) ON DELETE CASCADE`

### `messages`
- `id` TEXT PK
- `session_id` TEXT NOT NULL
- `role` TEXT NOT NULL
- `content_type` TEXT NOT NULL
- `content`
- `card_data`
- `metadata`
- `created_at`
- 约束：
  - `FOREIGN KEY(session_id) REFERENCES sessions(id) ON DELETE CASCADE`

### `loop_logs`
- `id` TEXT PK
- `session_id`
- `message_id`
- `round`
- `phase`
- `capability`
- `input_data`
- `output_data`
- `duration_ms`
- `created_at`
- 约束：
  - `FOREIGN KEY(session_id) REFERENCES sessions(id) ON DELETE CASCADE`

---

## 4.2 能力与配置域

### `mcp_servers`
- `id` TEXT PK
- `name` TEXT UNIQUE NOT NULL
- `command` TEXT NOT NULL
- `cwd`
- `description`
- `enabled` INTEGER（0/1）
- `source`（如 `db` / `file-import`）
- `created_at`
- `updated_at`

### `skills`
- `id` TEXT PK
- `name` TEXT UNIQUE NOT NULL
- `description` TEXT NOT NULL
- `body` TEXT
- `enabled` INTEGER（0/1）
- `source`
- `created_at`
- `updated_at`

### `llm_models`
- `id` TEXT PK
- `name` TEXT UNIQUE NOT NULL
- `provider`
- `base_url` TEXT NOT NULL
- `api_key`
- `is_default` INTEGER（0/1）
- `enabled` INTEGER（0/1）
- `extra_json`（扩展参数）
- `source`
- `created_at`
- `updated_at`
- 约束：
  - `UNIQUE INDEX idx_llm_models_single_default ON llm_models(is_default) WHERE is_default = 1`

### `prompts`
- `id` TEXT PK
- `prompt_key` TEXT UNIQUE NOT NULL
- `name` TEXT NOT NULL
- `content_md` TEXT NOT NULL（Markdown 内容）
- `description` TEXT
- `tags_json` TEXT（JSON 数组）
- `enabled` INTEGER（0/1）
- `source`
- `created_at`
- `updated_at`

### `agents`
- `id` TEXT PK
- `name` TEXT NOT NULL
- `agent_type` TEXT NOT NULL（`internal` / `external`）
- `description`
- `system_prompt`
- `base_url`（外部 Agent）
- `mcp_servers` JSON 字符串
- `agent_skills` JSON 字符串
- `allow_tools` JSON 字符串
- `deny_tools` JSON 字符串
- `max_rounds`
- `max_tool_phases`
- `timeout_seconds`
- `enabled` INTEGER（0/1）
- `source`
- `created_at`
- `updated_at`
- 约束：
  - `UNIQUE(agent_type, name)`

### `scheduled_tasks`
- `id` TEXT PK
- `name` TEXT UNIQUE NOT NULL
- `agent_name` TEXT NOT NULL
- `prompt` TEXT NOT NULL
- `interval_seconds` INTEGER NOT NULL
- `scheduler_mode`
- `task_tag`
- `enabled`
- `next_run_at`
- `last_run_at`
- `last_status`
- `last_output`
- `last_error`
- `created_at`
- `updated_at`

### `menus`
- `id` TEXT PK
- `menu_key` TEXT UNIQUE NOT NULL
- `name` TEXT NOT NULL
- `path`
- `component`
- `icon`
- `parent_key`
- `sort` INTEGER
- `visible` INTEGER（0/1）
- `enabled` INTEGER（0/1）
- `meta_json`
- `created_at`
- `updated_at`
- 约束：
  - `FOREIGN KEY(parent_key) REFERENCES menus(menu_key) ON DELETE SET NULL`
  - `CHECK(menu_key <> COALESCE(parent_key, ''))`

## 5. 索引设计

- `idx_channel_sessions_session_id`：按会话查渠道映射
- `idx_messages_session_created`：会话消息时间序读取
- `idx_scheduled_tasks_next_run`：调度任务按到期时间扫描
- `idx_menus_parent_sort`：菜单按父级与顺序查询
- `idx_llm_models_single_default`：保证默认模型唯一
- `idx_prompts_enabled`：提示词启用态快速检索

---

## 6. 关键一致性策略

- 会话删除时：
  - 级联删除 `messages`、`channel_sessions`、`loop_logs`
- 菜单层级：
  - 禁止自己挂自己
  - 父菜单不存在则拒绝写入
- 默认模型：
  - 仅允许一个默认模型

---

## 7. 数据来源与兼容

当前配置类数据采用“DB 主，文件兼容导入”策略：

- MCP：
  - 主数据源：`mcp_servers`
  - 兼容：当 DB 为空时可从 `.meowone/mcp.json` 导入

- Skill：
  - 主数据源：`skills`
  - 兼容：当 DB 为空时可从 `.meowone/skills/*` 导入

- LLM Model：
  - 主数据源：`llm_models`
  - 兼容：当 DB 为空时由环境变量导入默认模型

- Prompt：
  - 主数据源：`prompts`
  - 存储内容：`content_md`（Markdown 字符串）

---

## 8. 现阶段限制与后续建议

- SQLite 适合当前单实例部署；若多实例或高并发，建议迁移到 PostgreSQL。
- `agents` 与 `scheduled_tasks` 当前为逻辑关联（字符串字段），可在后续版本升级为显式外键。
- 建议补充数据库迁移机制（如 Alembic 或自研 migration runner），避免依赖 `CREATE TABLE IF NOT EXISTS` 的隐式演进。
